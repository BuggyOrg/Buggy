Semantic Programming

Natural Abstraction

Unlike other programming techniques, semantic programming creates abstraction layer naturally without limiting possible implementations. Buggy is an implementation of the Semantic Programming idea.
With Buggy it is possible to:

  • program on a high abstraction level
  • exchange implementations easily
  • avoid overhead associated with abstractions

Use computational power!

A main goal of Semantic Programming is to enable your PC to help you develop. Semantic Programming should avoid any boilerplate code and reduce the amount of work you need to put into developing. This means:

  • Write what you want to achieve not what the PC understands
  • Minimize syntactial overhead
  • Don't optimize manually!

Make Implementations comparable and exchangable!

To achieve the best in every application, programmers should be able to choose the best implementation on every possible level of abstraction.
Semantic Programming removes the implementation details from the program description and thus enables everyone to exchange every part of a program individually without loosing functionallity. This creates new possibilities when it comes to comparing programs. It is easier then ever to simply exchange one part of a program and to see its effects.
Furthermore programs can be build using quality or perfomance criteria to choose the best imeplementations at every stage.

Buggy Features

Independence

Buggy is a based on data. It is thus possible to create many different input languages that describe the problem in an appropiate way, even graphically.

With that we create programs that can run in the browser or on any platform. We can generate source code or compile native code and decide to run the program on multiple machines or on a single core.

Buggy can translate every program into any programming language if a few minimal instructions are defined.

Social Programming

Semantic Programming is independent of the implementation. Programs are described on a semantical level and not on a syntactial level. Thus implementations are exchangeable and Buggy uses this to support our Social Programming.
Social Programming is like a huge library. Everybody can upload their implementations and everyone else can use them. Buggy takes care of type conversion etc. To assure you find the correct implementation Buggy supports Semantic / Performance and other tests.
